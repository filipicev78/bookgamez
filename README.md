# Book Crossword

Book Crossword is a website where a user's reading history becomes a small daily crossword.

Users add books they have read, the system builds clue-answer pairs from those books, and each day the user gets one short crossword drawn from their personal library.

## Core idea

The product should feel like a tiny daily game, not study software.

The hook is:

> Your bookshelf comes back to you every morning as a tiny game.

## Product structure

The product has two main areas:

- `Daily Puzzle`
- `Books Read`

Everything else is secondary.

## MVP

Version 1 should include:

- one daily crossword
- 10 to 15 clues
- check, hint, reveal, and finish actions
- a completion recap
- book search by title
- adding books to a personal shelf
- book cards with `Ready` or `Building` style status
- a shared clue bank per canonical book
- basic bad clue reporting

## Product feel

The experience should be:

- light
- warm
- bookish
- game-first

It should feel closer to a small NYT-style game than a dashboard or study app.

## What not to build first

Do not treat these as v1 requirements:

- Kindle import
- Audible import
- social features
- leaderboards
- advanced stats
- multiple puzzle types
- mobile app

## Product vision

The product should make the user feel like their bookshelf comes back to them each morning as a short, satisfying game.
